UPDATED STEP 3D LIVE FEEDBACK IMPLEMENTATION PLAN
Clean implementation version


============================================================
0. PURPOSE
============================================================


Goal:
Build the first full live MEG neurofeedback self-test.


This is Step 3D:


live FieldTrip/Brainstorm MEG acquisition
→ live chunks
→ validated real-time processing core
→ live resting baseline
→ live trial z-scoring
→ green disk feedback
→ manual/success-based stopping
→ complete logs and audit files


This is not an offline Step 2D rehearsal.
Offline, simulated, and mock-live paths are development tools only.


Core project rule:
The data source can change, but the real-time processing core must remain the same.


Preserved interface:


[chunk, Source] = nf_get_meg_chunk(Source, RTConfig)
[Measure, RT] = nf_rt_process_chunk(chunk, RT, RTConfig)


The new live work should wrap around the validated core.
Do not rewrite the core algorithm.


============================================================
1. STEP 3D TARGET
============================================================


The first full live feedback delivery is successful when this command works in the MEG room:


nf_run_live_self_test


It should perform:


1. Start up project paths.
2. Build live config.
3. Finalize/validate config once.
4. Connect to live FieldTrip/Brainstorm buffer.
5. Confirm Fs = 2400 Hz.
6. Confirm channel labels/order.
7. Confirm sample count advances.
8. Prepare/validate live CombinedMatrix path.
9. Run live resting baseline.
10. Finalize and save Baseline.
11. Run live trial.
12. Compute theta power from a 2-second / 4800-sample window.
13. Compute ZRaw / ZClipped / ZSmoothed.
14. Map ZSmoothed to Shibata/Watanabe-style green disk size.
15. Display feedback.
16. Stop manually or by explicit success criterion.
17. Use 15–30 minute hard failsafe only as backup.
18. Save complete logs, traces, config, baseline, trial, stop reason, and session audit.
19. Close feedback display safely.


The first live delivery is not a polished participant experiment.
It is a technical self-test proving that the validated IIR/SOS real-time neurofeedback core can run on real 2400 Hz MEG chunks and drive actual visual feedback.


============================================================
2. LIVE SAMPLING AND WINDOW RULES
============================================================


Live MEG sampling rate:


Fs = 2400 Hz


Feedback power window:


PowerWindowSeconds = 2.0
PowerWindowSamples = 4800


Reason:


2.0 seconds * 2400 samples/second = 4800 samples


Live chunking:


ChunkSeconds = 0.2
ChunkSamples = 480


Reason:


0.2 seconds * 2400 samples/second = 480 samples


Window/chunk relationship:


PowerWindowSamples / ChunkSamples = 4800 / 480 = 10 chunks per feedback window


Initial circular buffer:


BufferSeconds = 2.0
BufferSamples = 4800


Rule:
The scientific feedback window is exactly 2 seconds = 4800 samples.
The circular buffer may later be increased to 2.5–3.0 seconds if delay correction or diagnostics require extra history, but the power window remains 2 seconds unless explicitly changed.


These timing rules apply to:
- live_fieldtrip
- mock_live_buffer


The mock-live path must not pass with timing that would fail in the MEG room.


Default strictness:


RTConfig.Debug.AllowNonLiveTimingInMock = false;


============================================================
3. CORE ALGORITHM ORDER — DO NOT CHANGE
============================================================


The live work must preserve the existing real-time algorithm order.


Required order:


1. Source returns chunk.Data in the configured live channel/correction state.


2. Source adapter may apply candidate CTF correction only if explicitly configured and logged.


3. Standard chunk is passed to:


   [Measure, RT] = nf_rt_process_chunk(chunk, RT, RTConfig)


4. Inside nf_rt_process_chunk:
   apply Spatial.CombinedMatrix / spatial projection.


5. Inside nf_rt_process_chunk:
   apply IIR/SOS filter statefully across chunks.


6. Inside nf_rt_process_chunk:
   append filtered projected data to circular buffer.


7. Inside nf_rt_process_chunk:
   extract latest 2-second / 4800-sample power window.


8. Inside nf_rt_process_chunk:
   reject invalid, warmup, gap, dropped, or artifact windows.


9. Inside nf_rt_process_chunk:
   compute target-band power / PowerPerSignal.


10. During resting:
    valid Measure.Power values update BaselineAcc.


11. At end of resting:
    BaselineAcc is trimmed/rejected/finalized into Baseline.


12. During trial:
    nf_rt_process_chunk computes z-score using Baseline.


13. Z-score order:
    ZRaw → ZClipped → ZSmoothed


14. Feedback remains outside nf_rt_process_chunk.


15. After nf_rt_process_chunk returns:
    nf_feedback_map_to_display maps Measure.ZSmoothed to FeedbackValue/radius.


16. Feedback display draws green disk.


17. Logger saves chunk metadata, Measure fields, traces, and session audit files.


Final live trial order:


live FieldTrip chunk
→ candidate source correction / channel selection
→ CombinedMatrix / spatial projection
→ stateful IIR/SOS
→ circular buffer
→ 2-second / 4800-sample theta power
→ Baseline z-score
→ ZSmoothed
→ map ZSmoothed to green disk radius
→ draw feedback
→ log everything


Hard rules:
- Do not move feedback drawing inside nf_rt_process_chunk.
- Do not assign FeedbackValue inside nf_rt_process_chunk.
- Do not branch nf_rt_process_chunk into a special live/single-channel path.
- Live fallback modes must still use the CombinedMatrix contract.

function RTConfig = nf_finalize_config(RTConfig)


Modes = nf_modes();


RTConfig.Paths.ProjectRoot = nf_project_root();


RTConfig.ChunkSamples = round(RTConfig.ChunkSeconds * RTConfig.Fs);
RTConfig.PowerWindowSamples = round(RTConfig.PowerWindowSeconds * RTConfig.Fs);
RTConfig.BufferSamples = round(RTConfig.BufferSeconds * RTConfig.Fs);


isLiveTimingMode = ...
    strcmp(RTConfig.Source.Mode, Modes.Source.LiveFieldTrip) || ...
    strcmp(RTConfig.Source.Mode, Modes.Source.MockLiveBuffer);


if isLiveTimingMode
    if strcmp(RTConfig.Source.Mode, Modes.Source.MockLiveBuffer) && ...
            isfield(RTConfig.Debug, 'AllowNonLiveTimingInMock') && ...
            RTConfig.Debug.AllowNonLiveTimingInMock
        % Explicit debug escape hatch.
    else
        assert(RTConfig.Fs == 2400, ...
            'Live/mock-live Fs must be 2400 Hz.');


        assert(RTConfig.ChunkSamples == 480, ...
            '0.2-second chunks at 2400 Hz must be 480 samples.');


        assert(RTConfig.PowerWindowSamples == 4800, ...
            '2-second window at 2400 Hz must be 4800 samples.');


        assert(RTConfig.BufferSamples >= RTConfig.PowerWindowSamples, ...
            'BufferSamples must be at least PowerWindowSamples.');


        assert(mod(RTConfig.PowerWindowSamples, RTConfig.ChunkSamples) == 0, ...
            'PowerWindowSamples must be an integer multiple of ChunkSamples.');
    end
end


% Session-scoped spatial validation.
% Acquisition-only sessions can run before CombinedMatrixPath is known.
% Spatial sessions require a valid matrix path if MatrixSource = Precomputed.
requiresSpatial = nf_session_requires_spatial(RTConfig);


% Spatial / CTF dependency.
matrixSource = RTConfig.Spatial.MatrixSource;


usesRealSpatialMatrix = ...
    strcmp(matrixSource, Modes.Spatial.MatrixSource.Precomputed) || ...
    strcmp(matrixSource, Modes.Spatial.MatrixSource.ComputeLive);


usesTechnicalFallback = ...
    strcmp(matrixSource, Modes.Spatial.MatrixSource.TechnicalFallback) || ...
    strcmp(matrixSource, Modes.Spatial.MatrixSource.TechnicalPlaceholder);


usesCTFCorrections = ...
    RTConfig.Source.CTF.ApplyChannelGains || ...
    RTConfig.Source.CTF.ApplyMegRefCorrection || ...
    RTConfig.Source.CTF.ApplyProjector;


% CTF res4 requirement:
% 1. CTF-dependent corrections require CTF res4.
% 2. Real spatial matrix paths require CTF res4 by default.
% 3. Clean technical fallback does not require CTF res4.
if isempty(RTConfig.Source.FieldTrip.RequireCTFRes4)
    if usesCTFCorrections
        RTConfig.Source.FieldTrip.RequireCTFRes4 = true;
    elseif usesRealSpatialMatrix
        RTConfig.Source.FieldTrip.RequireCTFRes4 = true;
    elseif usesTechnicalFallback
        RTConfig.Source.FieldTrip.RequireCTFRes4 = false;
    else
        RTConfig.Source.FieldTrip.RequireCTFRes4 = false;
    end
end


if usesTechnicalFallback && usesCTFCorrections
    warning(['Technical fallback is being used with CTF-dependent corrections enabled. ' ...
             'This is allowed only if CTF res4 is available and the correction state is logged.']);
end


% If fallback is meant to avoid CTF metadata, CTF-dependent corrections must be disabled.
if usesTechnicalFallback && ~RTConfig.Source.FieldTrip.RequireCTFRes4
    assert(~RTConfig.Source.CTF.ApplyChannelGains, ...
        'Technical fallback without CTF res4 cannot apply ChannelGains.');


    assert(~RTConfig.Source.CTF.ApplyMegRefCorrection, ...
        'Technical fallback without CTF res4 cannot apply MegRefCorrection.');


    assert(~RTConfig.Source.CTF.ApplyProjector, ...
        'Technical fallback without CTF res4 cannot apply projector.');
end


% CombinedMatrixPath validation is scoped to sessions that actually use spatial processing.
if strcmp(matrixSource, Modes.Spatial.MatrixSource.Precomputed)
    assert(isfield(RTConfig.Spatial, 'CombinedMatrixPath'), ...
        'Spatial.CombinedMatrixPath field is required for precomputed matrix mode.');


    if requiresSpatial
        assert(~isempty(RTConfig.Spatial.CombinedMatrixPath), ...
            ['Spatial.CombinedMatrixPath must be non-empty for sessions that require spatial processing. ' ...
             'Set a precomputed matrix path or switch MatrixSource to TechnicalFallback.']);


        assert(exist(RTConfig.Spatial.CombinedMatrixPath, 'file') == 2, ...
            'Spatial.CombinedMatrixPath does not point to an existing file.');
    end
end


% Trial failsafe single source of truth.
if ~isfield(RTConfig.Protocol.Trial, 'MaxFailsafeSeconds') || ...
        isempty(RTConfig.Protocol.Trial.MaxFailsafeSeconds)
    RTConfig.Protocol.Trial.MaxFailsafeSeconds = 30 * 60;
end


assert(RTConfig.Protocol.Trial.MaxFailsafeSeconds >= 15 * 60, ...
    'Trial hard failsafe must be at least 15 minutes.');


if RTConfig.Protocol.Trial.MaxFailsafeSeconds > 30 * 60
    warning('Trial hard failsafe is above 30 minutes. Confirm this is intentional.');
end


if isfield(RTConfig.Safety, 'MaxDurationSeconds') && ...
        ~isempty(RTConfig.Safety.MaxDurationSeconds)
    assert(RTConfig.Safety.MaxDurationSeconds == RTConfig.Protocol.Trial.MaxFailsafeSeconds, ...
        ['Do not maintain divergent trial failsafe fields. ' ...
         'Use Protocol.Trial.MaxFailsafeSeconds as source of truth.']);
end


RTConfig = nf_check_config(RTConfig);


end


Important:
nf_check_config runs before acquisition/resting/trial starts, not every 0.2 seconds.

============================================================
10. SPATIAL MATRIX ARCHITECTURE
============================================================


Decision:
For Step 3D, all live paths use the same downstream spatial contract.


Always use:


RTConfig.Spatial.Mode = Modes.Spatial.CombinedMatrix;


The path is selected only by:


RTConfig.Spatial.MatrixSource


Allowed sources:
- Modes.Spatial.MatrixSource.ComputeLive
- Modes.Spatial.MatrixSource.Precomputed
- Modes.Spatial.MatrixSource.TechnicalFallback
- Modes.Spatial.MatrixSource.TechnicalPlaceholder


Do not implement Path 3 by switching Spatial.Mode to single_channel or channel_average.
Path 3 must still produce Spatial.CombinedMatrix.


Reason:
This keeps nf_rt_process_chunk untouched and avoids special-case live processing.


------------------------------------------------------------
Path 1 — ComputeLive
------------------------------------------------------------


Configuration:


RTConfig.Spatial.Mode = Modes.Spatial.CombinedMatrix;
RTConfig.Spatial.MatrixSource = Modes.Spatial.MatrixSource.ComputeLive;


Meaning:
Build true live IPS CombinedMatrix from live header/source model.


Status:
Not implemented in this Step 3D branch unless Marc/Brainstorm provides the real computation script.


Behavior:
If ComputeLive is requested and no real computation path exists, nf_prepare_live_combined_matrix must error clearly.


Do not fake Path 1.


------------------------------------------------------------
Path 2 — Precomputed
------------------------------------------------------------


Configuration:


RTConfig.Spatial.Mode = Modes.Spatial.CombinedMatrix;
RTConfig.Spatial.MatrixSource = Modes.Spatial.MatrixSource.Precomputed;
RTConfig.Spatial.CombinedMatrixPath = '<path>';


Behavior:
Load precomputed CombinedMatrix.
Validate:
- matrix exists
- matrix has InputChannelNames
- matrix has correction-state metadata
- live channel order matches InputChannelNames
- correction state matches
- dimensions match corrected live chunk
- matrix contains no NaN/Inf


Log:
Spatial.MatrixSource = 'precomputed'
Spatial.IsTechnicalFallback = false
Spatial.IsIPS = true or unknown depending on metadata


Claim:
live technical self-test using precomputed IPS projection, if metadata supports IPS.


------------------------------------------------------------

function Results = nf_run_live_self_test()


startup;

RTConfig = nf_live_config();

DryRun = nf_source_dry_run(RTConfig);

end


if DryRun.Fs ~= 2400
    error('Live Fs mismatch: expected 2400 Hz.');
end


Source = nf_source_init(Modes.Source.LiveFieldTrip, [], RTConfig);
Spatial = nf_prepare_live_combined_matrix(Source, RTConfig);


if strcmp(RTConfig.Spatial.MatrixSource, Modes.Spatial.MatrixSource.ComputeLive) && ~Spatial.PreparedLive
    error('Live-computed CombinedMatrix requested but not implemented/prepared.');
end


if ~Spatial.ValidatedAgainstLiveHeader
    error('Spatial matrix was not validated against live header.');
end


Prepared = struct();
Prepared.Spatial = Spatial;
Prepared.SpatialHash = Spatial.Hash;
Prepared.LiveHeaderHash = Spatial.LiveHeaderHash;
Prepared.CreatedAt = now;


RTConfig.Session.Mode = Modes.Session.LiveResting;
RTConfig.Feedback.Mode = Modes.Feedback.None;
RestingResult = nf_run_live_resting(RTConfig, Prepared);


Baseline = RestingResult.Baseline;
if ~Baseline.Quality.Pass
    error('Live baseline failed quality check.');
end


RTConfig.Session.Mode = Modes.Session.LiveTrial;
RTConfig.Feedback.Mode = Modes.Feedback.LocalCircle;
RTConfig.Baseline.Path = RestingResult.BaselineFile;


TrialResult = nf_run_live_trial(RTConfig, Prepared);


Results = struct();
Results.DryRun = DryRun;
Results.Spatial = Spatial;
Results.Resting = RestingResult;
Results.Trial = TrialResult;
Results.Pass = TrialResult.Completed && Baseline.Quality.Pass;


end


Rationale:
- nf_run_live_self_test prepares Spatial once.
- Resting and trial reuse the same Prepared.Spatial.

- Standalone resting/trial still work when Prepared is absent.
