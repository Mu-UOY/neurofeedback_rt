======================================================================
0. DOCUMENT PURPOSE
======================================================================

This document defines the complete implementation path from the current state of
the neurofeedback_rt program to the next project endpoint:

A full production-equivalent FieldTrip simulation using recorded CTF/MEG data,
with the same downstream neurofeedback program that will later be used in the
live MEG room.

This document incorporates the current codebase state, all previously identified
missing work, and the final review refinements concerning representative early
loads, enforceable threshold locking, independently verified canonical hashing,
worst-case resting-extension capacity, calibrated producer-liveness thresholds,
explicit EOF classification, producer-termination semantics, robustness-test
status labeling, and reset recovery. It intentionally separates:

- early orchestration and performance-risk reduction;
- human scientific preparation and sign-off;
- matrix construction and scientific validation;
- complete-chain offline/streaming validation;
- producer lifecycle and continuous-stream handling;
- Psychtoolbox validation;
- strict final acceptance.

The plan ends when the complete recorded-CTF FieldTrip replay passes the strict
production-equivalence acceptance gate. It does not include the later live-room
session itself.

Plan control
------------

Plan.Version = '1.0'
Plan.Status = 'frozen'

After implementation begins, changes to this specification must occur through an
explicit new plan version and changelog entry. Implementation discoveries may
produce scoped amendments, but they must not silently weaken the acceptance gate
or mutate locked thresholds during a run.

1. FINAL GOAL
======================================================================

The final simulation data path is:

Recorded CTF/MEG dataset
-> ft_realtime_fileproxy
-> local FieldTrip buffer
-> the same live_fieldtrip reader intended for the MEG room
-> live header, sampling-rate, channel, and stream-state validation
-> the real precomputed full CombinedMatrix
-> IPS source signals with validated artifact suppression
-> residual-artifact detection and invalid-window rejection
-> stateful causal theta-band filtering
-> sliding-window theta-power estimation
-> resting baseline collection and quality validation
-> manual transition with continuous producer and explicit backlog discard
-> fresh trial processing state
-> trial z-scoring, clipping, and smoothing
-> real Psychtoolbox feedback
-> logging, stopping, validation, final reporting, and cleanup

The eventual live path is:

MEG acquisition room
-> ctf2t
-> FieldTrip buffer
-> the same downstream program used by the simulation

Everything after the FieldTrip buffer must be shared between simulation and live
operation.

Moving from the successful simulation to the MEG room should require only:

- source configuration changes;
- FieldTrip host and port changes;
- room-specific metadata;
- session-specific header/channel expectations;
- a session-compatible CombinedMatrix;
- Psychtoolbox display configuration;
- trigger/communication configuration when enabled.

No runtime or processing function may contain hidden machine-specific paths,
ports, channel lists, matrix paths, screen numbers, correction settings, or
room-specific assumptions.

======================================================================
3. FINAL PROCESSING CONTRACT
======================================================================

The final fixed spatial chain is:

CombinedMatrix
    = S_IPS
    * K_inverse
    * E_good
    * P_artifact
    * R_CTF
    * D_gain
    * C_channels

where:

C_channels
    maps the actual FieldTrip buffer channel order into the Brainstorm model
    order;

D_gain
    converts raw CTF counts into calibrated values only when the buffer has not
    already applied gains;

R_CTF
    converts the incoming compensation/reference state into the state expected
    by the Brainstorm inverse model only when this conversion has not already
    occurred;

P_artifact
    is the validated Brainstorm artifact-rejection projector;

E_good
    selects the exact ResultsMat.GoodChannel set;

K_inverse
    is the approved Brainstorm inverse imaging kernel;

S_IPS
    selects all source rows belonging to the approved IPS scout.

The online matrix contract is:

X_buffer:
    characterized FieldTrip sensor data
    [nInputChannels x nSamples]

Y_IPS:
    IPS source-row signals
    [nIPSSourceRows x nSamples]

Y_IPS = CombinedMatrix * X_buffer

All IPS rows remain separate through spatial projection and temporal filtering.
The power estimator then computes power per IPS row and averages row powers.

The project must not collapse all IPS source rows into one mean time series
before power unless that alternative measure is separately justified and
scientifically validated.

Fixed channel-space operations included in CombinedMatrix must not also be
applied in the source adapter.

Block-mean removal may remain a separately declared temporal preprocessing step
because it acts across samples rather than only across channel rows.


======================================================================
4. NON-NEGOTIABLE INVARIANTS
======================================================================

1. The final replay uses:
   Spatial.Mode = combined_matrix

2. The final replay requires:
   Spatial.IsIPS = true
   Spatial.IsTechnicalFallback = false

3. TechnicalFallback and TechnicalPlaceholder remain development tools only.

4. MatrixSource = compute_live does not need to be implemented for this
   milestone. The real matrix is prepared before resting and trial.

5. Resting and trial use the same matrix artifact and exact matrix hash.

6. The matrix is applied exactly once per accepted chunk.

7. Gains, CTF compensation, and artifact projector operations cannot be applied
   both outside and inside the matrix.

8. The source adapter cannot silently reinterpret the matrix input contract.

9. Resting and trial use the same live_fieldtrip source dispatcher and the same
   real-time core.

10. The final replay uses one continuous fileproxy producer across resting,
    transition, and trial.

11. The transition does not restart fileproxy in the final acceptance run.

12. During transition:
    - fileproxy continues advancing;
    - accumulated samples are discarded;
    - the skipped sample range is logged;
    - the source cursor jumps to the current live edge;
    - filter, circular-buffer, power-window, and z-smoothing state are reset.

13. Resting and trial begin with fresh temporal processing state.

14. Trial feedback remains suppressed until a complete fresh trial power window
    produces a valid measure.

15. All machine-specific and room-specific values are configuration-driven.

16. Required final validation sections cannot be SKIPPED.

17. The final Psychtoolbox run uses the real Psychtoolbox backend.

18. A development Psychtoolbox window may be used earlier, but it is explicitly
    labeled and rejected by the final acceptance gate.

19. The final replay runs at real-time speed and does not develop unbounded
    backlog.

20. Any unexpected buffer reset during the final acceptance run causes FAIL.

21. Producer readiness and continuous producer liveness are both monitored.

22. Exact artifact identity is verified through hashing; numerical equivalence
    is verified through tolerance-based comparison. These are separate concepts.

23. The offline equivalence reference uses the same causal one-directional
    filter behavior as the streaming path.

24. Scientific projector/scout approval is a human-reviewed gated milestone.

25. Thresholds are calibrated before the final run, locked, versioned, and
    hashed.

26. The final run must produce complete logs, provenance, cleanup evidence, and a
    plain-language event timeline.

27. At least one canonical hash golden vector is independently reproduced by a
    second implementation rather than generated only by the MATLAB serializer
    under test.

28. Canonical serializer edge-case tests run on the actual final stimulus
    computer.

29. Total dataset length is extracted from authoritative metadata, verified, and
    stored during stream characterization.

30. Producer-liveness thresholds are empirically calibrated, locked, and loaded
    from the threshold artifact.

31. EOF, healthy pacing jitter, producer stall, and producer process failure are
    distinct states with distinct stop reasons.

32. A robustness-test PASS means that expected failure detection and recovery
    behaved correctly. It can never be interpreted as a production-equivalent
    session PASS.

33. Artifact-detector uncertainty is itself pass/fail gated; an apparently good
    point estimate with inadequate evidence cannot pass.

34. Transition-capacity preflight uses the maximum possible configured resting
    endpoint, including the full allowed extension.

Rules

- first profile the existing contract;
- MATLAB dense multiplication should use optimized BLAS;
- do not assume sparse is faster;
- compare dense and sparse only if sparsity is genuine;
- MEX/GPU/parallel redesign is considered only if measured timing requires it.

Early gate
----------

Repeat gate after real matrix construction
------------------------------------------

validating disconnected workloads.

This is not the final timing validation because it excludes:

- FieldTrip read latency;
- producer behavior;
- full logging;
- Psychtoolbox flips;
- end-to-end consumer lag.

3. R_CTF

Build the verified compensation/reference conversion.

Requirements:

- correct MEG/reference channel sets;
- verified coefficients;
- identity when no conversion is needed;
- unsupported grade transitions fail;

4. P_artifact

Build the approved projector.

Requirements:

- approved components only;
- component identity saved;
- symmetry checked;
- idempotence checked;
- selected components annihilated;
- identity allowed only with explicit scientific approval.

5. E_good

Build selector from ResultsMat.GoodChannel.

Requirements:

- do not recompute GoodChannel;
- preserve exact order;
- kernel column count must match.

6. K_inverse

Load ResultsMat.ImagingKernel.

Requirements:

- valid dimensions;

7. S_IPS

Convert approved IPS vertices into actual source rows.

Requirements:

- use nComponents and GridAtlas as needed;
- nonempty;
- valid row indices;
- preserve all IPS rows;
- save source rows and scout metadata.

8. Compose

CombinedMatrix
    = S_IPS

    * E_good
    * P_artifact
    * R_CTF
    * D_gain
    * C_channels

2. canonically serialize the defined hashable payload, excluding the stored
   Thresholds.Hash field;
3. recompute its SHA-256 hash;
4. require the recomputed hash to equal Thresholds.Hash;
5. require the same hash to equal
   RTConfig.Validation.RequiredThresholdHash;
6. require Thresholds.Locked = true;
7. reject any config-level threshold override;
8. copy the exact verified threshold artifact into the session bundle.

Rules
-----

- thresholds are not derived from the final run;
- thresholds are not modified during final acceptance;
- any change requires a new version and hash;
- thresholds distinguish numerical identity from scientific acceptability;
- thresholds are strict enough to detect real processing differences;
- thresholds are not set so tightly that normal floating-point noise causes
  meaningless failure;
- a failed resting rehearsal is not permission to loosen thresholds;
- threshold changes require documented scientific rationale, repeated
  calibration, a new version, a new hash, and rerunning dependent validation;
- clean-window rejection is checked for systematic concentration around warm-up
  or phase boundaries rather than true contamination;
- expert-reviewed real clean data are the primary artifact-detector calibration
  source;
- synthetic clean data may supplement targeted tests but cannot replace
  real-clean-session validation;
- insufficient clean windows fails calibration;
- uncertainty wider than the locked maximum fails calibration;
- an upper false-positive confidence bound above the locked maximum fails
  calibration;
- ProducerLiveness thresholds are derived from multiple healthy real-time replay
  runs and are not selected arbitrarily;
- liveness thresholds are part of the locked artifact and cannot be changed by
  the final runner.

Robustness-test result semantics
--------------------------------

Every deliberately destructive robustness test records:

TestResult.TestClass = 'robustness'
TestResult.ExpectedProductionFailure = true
TestResult.RecoveryBehaviorPassed

A deliberately stuck producer followed by successful force-kill may report:

RecoveryBehaviorPassed = true
TestStatus = 'PASS'
ProductionEquivalent = false

The report must state:

'This isolated robustness test passed because the expected failure was detected
and cleaned up correctly. It is not a production-equivalent session PASS.'

A nominal final run requiring force-kill still fails the producer lifecycle
requirement. Cleanup completion and graceful producer behavior are reported
separately.

Final acceptance rule
---------------------

Recovery behavior may be tested separately, but the final PASS run must not
contain:

- producer stall;
- unexpected reset;
- unexplained gap;
- dropped chunk;
- unbounded backlog;
- forced producer kill caused by normal-run failure.
